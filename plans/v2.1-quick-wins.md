# V2.1 Quick Wins: Export, Bulk Enrich, Graph Filtering

## Objective

Add three high-impact, low-effort features to Nexus:
1. **Export/Share** ‚Äî export a project or global graph as JSON or Markdown
2. **Bulk Enrich** ‚Äî one-click enrichment of all unenriched concepts
3. **Graph Filtering** ‚Äî filter graph view by expertise level (known_well / seen)

## Acceptance Criteria

### Export
- [ ] `GET /api/graph/export?format=json&project_id=...` returns downloadable JSON
- [ ] `GET /api/graph/export?format=markdown&project_id=...` returns formatted Markdown
- [ ] Toolbar "export" button opens format picker, triggers browser download
- [ ] Markdown includes concept name, category, description, connections

### Bulk Enrich
- [ ] `POST /api/concepts/enrich-bulk?project_id=...` starts background enrichment
- [ ] `GET /api/enrich-status` returns progress (e.g. `"enriching (3/12)"` or `null`)
- [ ] Toolbar "enrich all" button with live progress indicator
- [ ] Only enriches concepts missing descriptions (per existing decision)
- [ ] Graceful per-concept failure (continue enriching others)

### Graph Filtering
- [ ] Concepts include `expertise_level` field in API responses (`known_well` or `seen`)
- [ ] LeftSidebar shows expertise filter buttons below categories
- [ ] GraphView filters nodes by expertise level (same pattern as category filter)
- [ ] Filters compose: category + expertise can be active simultaneously

## Steps

### Backend

1. **Export endpoint** (`routes.py`) ‚Äî add `GET /api/graph/export` that reuses
   `list_concepts` + `get_all_edges`, formats as JSON (with Content-Disposition
   header) or Markdown string. ~30 LOC.

2. **Bulk enrich endpoint** (`routes.py`) ‚Äî add `POST /api/concepts/enrich-bulk`
   with background task that loops unenriched concepts (same pattern as scan in
   `routes_projects.py:109-119`). In-memory `_enrich_status` dict for progress.
   Add `GET /api/enrich-status` for polling. ~40 LOC.

3. **Expertise level on concepts** (`server.py`) ‚Äî add `expertise_level` computed
   field to `concept_dict()`. Logic: has description + has embedding + appears
   in edges ‚Üí `known_well`, else `seen`. Requires passing edge set or computing
   inline. ~10 LOC.

### Frontend

4. **Export button + download** (`App.tsx`) ‚Äî add "export" toolbar Shortcut.
   On click, fetch `/graph/export?format=markdown&project_id=...` and trigger
   browser download via Blob URL. ~15 LOC in App.tsx.

5. **Bulk enrich button + status** (`App.tsx` + `useApiExtra.ts`) ‚Äî add "enrich
   all" toolbar Shortcut. Add `useBulkEnrich` mutation + `useEnrichStatus` poll
   hook. Show progress text next to button. ~25 LOC across files.

6. **Expertise filter** (`LeftSidebar.tsx` + `App.tsx` + `GraphView.tsx`) ‚Äî add
   `expertiseFilter` state in App, pass to LeftSidebar (new buttons) and
   GraphView (filter in `buildElements`). ~30 LOC across files.

### Tests

7. **Backend tests** ‚Äî test export JSON/Markdown format, bulk enrich status
   tracking, expertise_level computation.
